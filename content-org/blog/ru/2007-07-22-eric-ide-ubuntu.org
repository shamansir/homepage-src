#+title: Eric IDE 4.0.1 на Ubuntu 7.04
#+datetime: 22 Jul 2007 18:05
#+tags: eric-ide python ubuntu
#+hugo_section: blog-ru

[[http://www.die-offenbachs.de/eric/index.html][Eric]] -- очень даже
хорошее IDE под [[http://www.python.org/][Python]]. И
[[http://www.die-offenbachs.de/eric/eric-news.html][не далее как вчера]]
вышла версия
[[http://sourceforge.net/project/showfiles.php?group_id=119070&package_id=233329][4.0.1]],
в репозиториях же на данный момент лежит версия 3.9. и захотелось мне
вспомнить старые добрые времена и собрать этот пакет. Представляю вам на
всякий случай листинг того, что делать -- чтобы не возвращаться обратно,
выясняя что нужно поставить еще что-то или разбираясь в ошибках
компиляции, которые меня посещали :). Кое-где -- ориентировка на
последние версии :). Так как устанавливалось на чистой практически
убунте -- все должно быть адекватно... если что-то упустил -- прошу
сообщать.

/Используется:/

- [[http://www.die-offenbachs.de/eric/index.html][eric]] 4.0.0.1
- [[http://www.python.org/][Python]] 2.5.1
- [[http://trolltech.com/products/qt][Qt]] 4.2.3 +
  [[http://www.riverbankcomputing.co.uk/pyqt/index.php][PyQt]] v4.1 +
  [[http://www.riverbankcomputing.co.uk/qscintilla/index.php][QScintilla]]
  2
- [[http://www.riverbankcomputing.co.uk/sip/index.php][SIP]] 4.6
- [[http://gcc.gnu.org/][G++]] 3.4 (наверняка можно взять и поздний)

итак, переходим в какой-нибудь каталог для сборки и приступаем:

#+begin_example
$ cd ~/distr-temp/
#+end_example

берем
[[http://sourceforge.net/project/showfiles.php?group_id=119070&package_id=233329][сурсы]]
с sourceforge‘a:

#+begin_example
$ wget http://mesh.dl.sourceforge.net/sourceforge/eric-ide/eric4-4.0.1.tar.gz
#+end_example

устанавливаем всяческие библиотеки для связки
[[http://www.riverbankcomputing.co.uk/pyqt/index.php][PyQt]] (в
зависимости от настроек =apt-get='а может понадобиться установочный CD):

#+begin_example
$ sudo apt-get install python2.5-dev
$ sudo apt-get install python-qt4
$ sudo apt-get install libqt4-dev
$ sudo apt-get install python-qt4-dev
#+end_example

устанавливаем компилятор =g++= и делаем на него ссылку =/usr/bin/g++=:

#+begin_example
$ sudo apt-get install g++-3.4
$ sudo ln -s /usr/bin/g++-3.4 /usr/bin/g++
#+end_example

скачиваем и устанавливаем последнюю на данный момент версию
[[http://www.riverbankcomputing.co.uk/sip/index.php][SIP]] -- он
позволяет библиотекам, написанным на C[++] вести себя как модули
[[http://www.python.org/][Python]]:

#+begin_example
$ wget http://www.riverbankcomputing.com/Downloads/sip4/sip-4.6.tar.gz
$ tar -xvzf ./sip-4.6.tar.gz
$ cd ./sip-4.6.tar.gz
$ python ./configure.py
$ make
$ sudo make install
$ cd ..
#+end_example

скачиваем и устанавливаем последнюю на данный момент версию
[[http://www.riverbankcomputing.co.uk/qscintilla/index.php][QScintilla]]
-- Qt-порт компонента для редактирования исходных кодов (устанавливаем
версию для Qt4):

#+begin_example
$ wget http://www.riverbankcomputing.com/Downloads/Snapshots/QScintilla2/QScintilla-gpl-2-snapshot-20070709.tar.gz
$ tar -xvzf ./QScintilla-gpl-2-snapshot-20070709.tar.gz
$ cd ./QScintilla-gpl-2-snapshot-20070709/Qt4
$ qmake qscintilla.pro
$ sudo make
$ sudo make install
#+end_example

теперь нужно установить связи на
[[http://www.riverbankcomputing.co.uk/qscintilla/index.php][QScintilla]]
для [[http://www.python.org/][Python]] -- они находятся собственно в
директории Python:

#+begin_example
$ cd ../Python
$ python ./configure.py
$ make
$ sudo make install
#+end_example

ну а теперь очередь собственно
[[http://www.die-offenbachs.de/eric/index.html][eric]]:

#+begin_example
$ cd ../../eric4-4.0.1/
$ sudo python install.py
#+end_example

при установке нужно сказать ему директорию где лежит Qt:
=/usr/share/qt4=.

Ну вот собственно и все. Набираем =eric4= и, если хочется, радуемся :).
